              Linux basic commands
            </h1>

          <div class="crayons-article__body text-styles spec__body" data-article-id="995063" id="article-body">
            <p>Linux is an extremely common operating system. 90% of all cloud computing systems run on Linux. Most servers and technology-embedded devices run on Linux. It’s the Linux that manages and orchestrates the computing power in most of the cloud. </p>

<h3>
  <a name="linux" href="#linux">
  </a>
  Linux
</h3>

<p>Linux is an operating system's kernel. It was actually created by Linus Torvalds from Scratch. Linux is free and open-source, that means that you can simply change anything in Linux and redistribute it in your own name! There are several Linux Distributions, commonly called “distros”. </p>

<ul>
<li><p>Ubuntu Linux</p></li>
<li><p>Red Hat enterprise Linux</p></li>
<li><p>Linux Mint</p></li>
<li><p>Debian</p></li>
<li><p>Fedora</p></li>
</ul>

<p>*<em>Linux Shell *</em> is basically a shell is a program that receives commands from the user and gives it to the OS to process, and it shows the output. </p>

<h3>
  <a name="basic-commands" href="#basic-commands">
  </a>
  Basic Commands
</h3>

<ul>
<li><p><strong><code>man</code></strong> - The man command is a built-in manual for using Linux commands (short for manual page) . The man page includes a command description, applicable options, flags, examples, and other informative sections. <code>man ls</code> gives info about ls command</p></li>
<li><p><code>clear</code>- It is a simple yet frequently used command which basically clears the screen.</p></li>
<li><p><code>sudo</code> -  Just like we have <code>run as administrator</code> in Windows, <code>sudo</code> allows you to run a command as Super User ( admin privileges).</p></li>
<li><p><code>mkdir</code>- this will allow us to create a folder in our system. With the following command we can create a folder in our home directory called LinuxCommands <code>mkdir LinuxCommands</code>.</p></li>
<li><p><code>cd</code>- allows us to change directory, so for us to move into our newly created directory we can do this with <code>cd LinuxCommands</code>. If we want to get back to where we started we can use <code>cd ..</code> ( one Level back)</p></li>
<li><p><code>pwd</code>- print working directory, <code>pwd</code> gives us the print out of the current directory.</p></li>
<li><p><code>touch</code>- We can create files using the touch command if we were to run <code>touch srinivas.html</code>this would create a file named srinivas.html</p></li>
<li><p><code>ls</code>- This is going to list the all the files and folders in the current directory. <code>ls -a</code> lists all files along the hidden files.</p></li>
<li><p><code>mv</code>- This is going to allow you to move your files.  <code>mv LinuxCommands Basics</code> will move your file to the Basics folder.</p></li>
<li><p><code>cp</code>- If I just want to copy files from one folder to another, simply put its very similar to the <code>mv</code> command but we use <code>cp</code> .</p></li>
<li><p><code>rm</code>-simply <code>rm Srinivas.html</code> will remove the file. We will also use quite a bit <code>rm -R</code> which will recursively work through a folder or location. </p></li>
<li><p><code>rmdir</code> - allows for us to remove the directory, if we run <code>rmdir LinuxCommands</code> , the folder will be removed.</p></li>
<li><p><code>echo</code>- The echo command  prints out arguments as the standard output. <code>echo "Hello World"</code> prints out - Hello world on screen</p></li>
<li><p><code>cat</code>- We can use <code>cat</code> to see the contents inside the file. </p></li>
<li><p><code>grep</code>-If we have a large file and we don't want to read every line then <code>grep</code> is your friend, this will allow us to search your file for a specific word using <code>cat main.go | grep "Println"</code> . It outputs all the lines that contain "println".</p></li>
<li><p><code>history</code>-used to find out all those commands we ran previously.</p></li>
<li><p><code>useradd</code>- If we want to add new users to our system, we can do this using <code>useradd</code> as a sudo operation. we can add a new user with <code>sudo useradd NewUser</code></p></li>
<li><p><code>groupadd</code>- Creating a group also a sudo operation and we can use <code>sudo groupadd newgroup</code></p></li>
<li><p><code>usermod</code>- is used to change the properties of a user. If we want to add our new user to the group we created, we can do this by running <code>sudo NewUser usermod -a -G newgroup</code>. -a is add and -G is group name.</p></li>
</ul>

<h3>
  <a name="permissions" href="#permissions">
  </a>
  Permissions
</h3>

<ul>
<li>
<code>ls -l</code> = Print files in a long listing format, you can see ownership and permissions of the file</li>
</ul>

<p><strong>Ownership:</strong></p>

<ul>
<li>
<code>sudo chown [username]:[groupname] [filename]</code> = Change ownership</li>
<li>
<code>sudo chown tom:admin test.txt</code> = Change ownership of 'test.txt' file to 'tom' and group 'admin'</li>
<li>
<code>sudo chown admin test.txt</code> = Change ownership of 'test.txt' 'admin' user</li>
<li>
<code>sudo chgrp devops test.txt</code> = Make 'devops' group owner of test.txt file</li>
</ul>

<p><strong>Possible File Permissions (Symbolic):</strong></p>

<ul>
<li>r = Read</li>
<li>w = Write</li>
<li>x = Execute</li>
<li>'-' = No permission</li>
</ul>

<p><strong>Change File Permissions for different owners</strong></p>

<p>File Permissions can be changed for:</p>

<ul>
<li>u = Owner</li>
<li>g = Group</li>
<li>o = Other (all other users)</li>
</ul>

<p>Minus (-) removes the permission</p>

<ul>
<li>
<code>sudo chmod -x api</code> = Takes 'execute' permission away for 'api' folder from all owners</li>
<li>
<code>sudo chmod g-w config.yaml</code> = Takes 'write' permission away for 'config.yaml' file from the group </li>
</ul>

<p>Plus (+) adds permission</p>

<ul>
<li>
<code>sudo chmod g+x config.yaml</code> = Add 'execute' permission for 'config.yaml' file to the group </li>
<li>
<code>sudo chmod u+x script.sh</code> = Add 'execute' permission for 'script.sh' file to the user </li>
<li>
<code>sudo chmod o+x script.sh</code> = Add 'execute' permission for 'script.sh' file to other </li>
</ul>

<p>Change multiple permissions for an owner</p>

<ul>
<li>
<code>sudo chmod g=rwx config.yaml</code> = Assign 'read write execute' permissions to the group</li>
<li>
<code>sudo chmod g=r-- config.yaml</code> = Assign only 'read' permission to the group</li>
</ul>

<p>Changing permissions with numeric values</p>

<p><em>Set permissions for all owners with 3 digits, 1 digit for each owner</em> <a href="https://docs.oracle.com/cd/E19455-01/805-7229/6j6q8svd8/">Absolute vs Symbolic Mode</a></p>

<ul>
<li>0 = No permission</li>
<li>1 = Execute</li>
<li>2 = Write</li>
<li>3 = Execute + Write</li>
<li>4 = Read</li>
<li>5 = Read + Execute</li>
<li>6 = Read + Write</li>
<li>7 = Read + Write + Execute
&lt;!-- --&gt;</li>
<li>
<code>sudo chmod 777 script.sh</code> = rwx (Read, Write and Execute) permission for everyone for file 'script.sh'</li>
<li>
<code>sudo chmod 740 script.sh</code> = Give user all permissions (7), give group only read permission (4), give other no permission (0)</li>
</ul>
